# Kaspi Pay: автоматизация оплаты для ATMOS

Сейчас оплата проверяется вручную: покупатель нажимает «Я оплатил», администратор проверяет платёж в Kaspi Pay и подтверждает заказ в админке. Билеты выдаются только после подтверждения.

## Варианты автоматизации

### 1. Официальное партнёрство с Kaspi (для приёма платежей на сайте)

- **Где:** [kaspi.kz/webpay/partnership](https://kaspi.kz/webpay/partnership)
- **Требования:** ИП/ТОО, расчётный счёт, ЭЦП. Рассмотрение заявки 6–15 рабочих дней.
- **Результат:** доступ к приёму платежей и, при наличии, к API/вебхукам для автоматического подтверждения оплаты. Нужно уточнять у поддержки Kaspi, выдают ли после одобрения API для мерчантов.

### 2. Сторонний сервис ApiPay.kz (REST API для Kaspi Pay)

- **Сайт:** [apipay.kz](https://apipay.kz)
- **Суть:** создание счетов по API, уведомления об оплате (webhook). Комиссия 0%, есть платные тарифы (от 10 000 ₸/мес).
- **Подключение:** регистрация на ApiPay.kz → написать в WhatsApp (+7 708 516 74 89) для подключения Kaspi Business с правами «Кассира» → получить API-ключ → интегрировать на сайт.
- **Плюсы:** можно автоматически создавать счета с суммой и комментарием (номер заказа) и получать вебхук об оплате, затем помечать заказ оплаченным и открывать билеты на странице чека.

### 3. Текущая схема (без API)

- Покупатель переходит по ссылке Kaspi Pay, вводит сумму, платит.
- Нажимает «Я оплатил» на сайте → в Telegram приходит уведомление администратору.
- Администратор проверяет платёж в приложении Kaspi Pay и нажимает «Подтвердить» в админке.
- Покупатель открывает сохранённую ссылку на заказ (`/booking/receipt/[orderId]`) и скачивает билеты после подтверждения.

Риск: кто-то может нажать «Я оплатил» без оплаты; в этом случае заказ так и остаётся в статусе «Ожидает», билеты не выдаются, места через 30 минут освобождаются. То есть обмана нет — билеты появляются только после ручного подтверждения.
